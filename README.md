# BirdLog 观鸟日记 — 产品需求文档

> **项目性质：** 个人日记本型观鸟记录应用，兼具正念/情绪落地功能  
> **目标用户：** 开发者本人，中国观鸟爱好者  
> **开发语言：** 中文为主界面语言  

---

## 技术要求

### 项目初始化
- 使用 Vite + React + TypeScript 创建项目
- 样式方案：Tailwind CSS 或 CSS Modules（不要行内样式）
- 组件拆分为独立文件，不要写巨型单文件组件
- 数据持久化使用 `localStorage`（后续可升级 IndexedDB）
- 存储 key 统一用 `birdlog-` 前缀

## 二、数据模型

### 2.1 事件（Event）

每次出门观鸟 = 一个事件，是最核心的数据单位。

```typescript
interface Event {
  id: string;             // 唯一标识
  date: string;           // YYYY-MM-DD
  startTime?: string;
  endTime?: string;
  isAbroad: boolean;
  country?: string;       // 仅国外
  province?: string;      // 仅国内，下拉选择
  city?: string;          // 市/区
  location: string;       // 具体地点名
  companions: string[];   // 同行人名字
  records: BirdRecord[];  // 鸟单
  journal: string;        // 观鸟流水账
  journalMedia: MediaRef[];
  createdAt: string;      // ISO string
}
```

### 2.2 鸟种记录（BirdRecord）

```typescript
interface BirdRecord {
  birdId: number;         // 对应鸟种数据的 id 字段
  birdZh: string;         // 冗余存储中文名
  qty: number;            // 数量，默认 1
  star: boolean;          // 标星 = "这次观察值得一记"
  gender: "未知" | "雄" | "雌";
  behaviors: string[];    // 基础行为 + 自定义行为
  customBehaviors: string[]; // 该记录上用户新建的自定义行为
  note: string;           // 鸟种笔记
  photos: MediaRef[];
  audio: MediaRef[];
}
```

### 2.3 媒体（MediaRef）

```typescript
interface MediaRef {
  id: string;
  type: "photo" | "audio";
  uri: string;            // base64 data URL
  timestamp: string;
  caption?: string;
}
```

### 2.4 鸟种静态数据

来源文件：`china_birds.json`，每条结构：

```typescript
interface Bird {
  id: number;   // IOC 序号
  s: string;    // 学名 Scientific name
  e: string;    // 英文名 English name
  z: string;    // 中文名
  p: string;    // 全拼（如 zongbeibolao）
  i: string;    // 拼音首字母（如 zbbl）
}
```

当前收录 1140 种中国鸟种。全球数据（11194种）在 `bird_list_ioc14_1.json` 中备用。

### 2.5 朋友（Companion）

```typescript
interface Companion {
  name: string;           // 姓名，作为唯一标识
  note?: string;          // 用户填写的备注（如"大学同学"）
  photo?: string;         // 头像，base64 data URL
}
```

同行次数和首次同行日期从事件数据中实时计算，不冗余存储。

### 2.6 省份常量（34项）

```
北京 天津 河北 山西 内蒙古 辽宁 吉林 黑龙江
上海 江苏 浙江 安徽 福建 江西 山东 河南
湖北 湖南 广东 广西 海南 重庆 四川 贵州
云南 西藏 陕西 甘肃 青海 宁夏 新疆 香港 澳门 台湾
```

### 2.7 行为常量（6项基础 + 自定义）

基础：求偶\筑巢与育雏\领地行为\觅食\社交\休息

用户可在任意鸟种记录上新建自定义行为。自定义行为与成就系统关联（详见成就章节）。

## 三、页面详细设计

### 3.1 首页

**问好统计：**
- "你好 🐦"
- "你已经观鸟 X 次，观察到 X 种鸟，结识了 X 位鸟人"

**新建记录按钮：** 只有一个入口按钮。点击后进入记录流程，日期默认当天。需要补录时用户手动修改日期即可。

**情绪落地入口（占位，内容后续补充）：**
- "嘿朋友，你还有力气出门吗？"
- 两个选项："去户外" / "没力气"

**回忆卡片：** 随机展示一条星标过的鸟种笔记（有星标事件时才显示）。

### 3.2 记录页（全屏流程）

从首页"新建记录"进入，覆盖整个屏幕（不是Tab页面）。

**顶部栏：** 左侧返回按钮 + "新建记录"文字，右侧"完成"按钮（鸟单不为空时可点击）。

**信息区卡片：**
- 🇨🇳中国 / 🌏国外 切换按钮
- 中国模式：省份下拉选择（34项）+ 市/区文本输入 + 观测地点文本输入
- 国外模式：国家文本输入 + 地点文本输入
- 日期选择器（默认当天）
- 同行人：已添加的显示为标签（可删除），"+ 同行人"按钮新建。若已有历史同行人，显示"常用"快捷标签。关联到朋友页面,新建同行人加入朋友

**鸟种搜索框：**
- 占位文字："搜索：中文 / 拼音首字母 / 英文 / 学名"
- 搜索逻辑（五维度同时匹配）：
  - 中文名包含输入内容
  - 拼音首字母以输入内容开头（如 zbbl → 棕背伯劳）
  - 全拼以输入内容开头
  - 英文名包含（不区分大小写）
  - 学名包含（不区分大小写）
- 结果下拉列表最多 20 条
- 已添加的鸟种在结果中标灰显示"已添加"

**统计条：** "本次记录 X 种鸟，X 种标星"

**鸟单列表（核心交互）：**

每条记录卡片包含：
- **主行：** ★星标切换 | 中文名（点击展开详情）| 数量（点击弹出数字输入框）| 📷照片按钮 | 🎤音频按钮 | ✕删除
- **副行：** 英文名（斜体，缩进到星标按钮后方，独立一行，不与其他元素挤在同一行）

点击中文名展开详情面板：
- **性别：** 未知 / 雄 / 雌（单选按钮组）
- **行为：** 6个基础行为标签（多选）+ 已创建的自定义行为 + "自定义"新建按钮
- **鸟种笔记：** 多行文本框
- **照片/音频缩略图预览**（添加后显示在这里）

**数量输入：** 不用 +/- 按钮（容易误触）。显示当前数字，点击后变为输入框，输入数字后失焦或回车确认。

**照片功能：**
- 点击📷 → `<input type="file" accept="image/*">` 选择或拍照
- 照片压缩为合理尺寸后存为 base64
- 缩略图预览在展开详情中

**音频功能：**
- 点击🎤 → 开始录音（MediaRecorder API）或选择音频文件
- 显示为可播放的音频条

**悬浮流水账按钮：**
- 固定在右下角的圆形按钮（笔图标）
- 有内容时按钮上显示小圆点提示
- 点击弹出底部半屏面板
- 面板内容：文本框 + 可插入本事件中已添加的照片/音频
- 点击面板外区域收回

**完成流程：** 点"完成" → 进入预览页 → 确认后点"保存" → 数据写入 localStorage → 返回首页。

### 3.3 预览/汇总页

- **统计卡片（渐变色）：** 日期、总种数、标星数、同行者、地点
- **事件记录：** 流水账全文（含内联照片/音频）
- **鸟种笔记汇总：** 可展开的折叠栏，默认收起。展开后每条显示：鸟种名 + 行为标签 + 笔记文字
- **鸟单：** 星号列固定宽度（保证有星和没星的鸟中文名对齐），格式：★ 棕背伯劳 *Lanius schach* ×2

### 3.4 事件列表页

每个事件一张卡片：地点、日期、鸟种数量、同行者。点击进入详情页（复用汇总页的视图）。

按时间倒序排列（最近的在上面）。

### 3.5 成就页

**预设成就：**

| 成就名 | 触发条件 | 图标 |
|--------|----------|------|
| 初识飞羽 | 累计记录 ≥ 1 种鸟 | 🪶 |
| 十鸟临门 | 累计记录 ≥ 10 种 | 🐦 |
| 半百观鸟人 | 累计记录 ≥ 50 种 | 🦅 |
| 百鸟归巢 | 累计记录 ≥ 100 种 | 🦉 |
| 值得一记 ×10 | 累计标星 ≥ 10 次 | ⭐ |
| 值得一记 ×50 | 累计标星 ≥ 50 次 | 🌟 |
| 省级观鸟人 | 在 ≥ 5 个不同省份观鸟 | 🗺️ |
| 全国行者 | 在 ≥ 15 个不同省份观鸟 | 🌏 |
| 独行侠 | 累计 ≥ 10 次独自观鸟（无同行人） | 🚶 |
| [省名]达人 | 在某省记录 ≥ 50 种 | 🏅 |
| 风雨同路 | 与同一人同行 ≥ 10 次 | 🤝 |
| 全行为大师：[鸟名] | 某鸟种的所有行为均记录过（见下方说明） | 🎯 |

**全行为收集的特殊逻辑：**
- 达成条件：6种基础行为全部记录过 **并且** 该鸟种上出现过的所有自定义行为也被记录过
- 达成后：该鸟种在"我的"页面的时间轴中名字显示为金色

**成就显示：**
- 未达成：灰色卡片 + 进度条（如 3/10）
- 达成瞬间：撒花动画 + 通知（在事件保存时触发检测）
- 已达成：彩色卡片 + "已达成 ✓"

**自定义成就：** 用户可自行创建,根据鸟种,地区,友人等设置,类似苹果快捷指令的自由度。

### 3.6 我的页面

两个子Tab切换：**鸟种时间轴** | **朋友**

**鸟种时间轴：**
- 按首次观测日期从近到远排列
- 同一日期下的鸟种分组显示
- 每行格式：`序号 中文名 学名(斜体) ★★★`,当★大于五,显示为★*n(n>5)
- 序号 = life list 排序（第几个新种）
- 星星数量 = 所有事件中该鸟种被标星的总次数
- 全行为收集达成的鸟种 → 名字显示金色
- 示例：
```
2026年3月1日
156 雪雁 Anser caerulescens ★
155 白鹤 Leucogeranus leucogeranus
┊
2026年1月10日
2 白头鹎 Pycnonotus sinensis ★★★★
1 棕背伯劳 Lanius schach
```

**朋友列表：**
- 每位朋友一张卡片：头像照片（如有）+ 姓名 + 备注 + 同行次数 + 首次同行日期
- 同行次数和首次日期从事件数据实时计算
- 点击卡片 → 展开编辑：可修改备注、上传/更换照片

## 四、数据导入（后续实现）

### 中国观鸟记录中心

导出格式 TSV，字段：
```
报告编号 | 观测开始时间 | 观测结束时间 | 中文名 | 拉丁名 |
省 | 州/市 | 区/县 | 观测点名称 | IUCN受胁级别 |
国家保护等级 | CITES保护等级 | 鸟种数量
```

导入逻辑：
1. 按"报告编号"分组，每组 = 一个事件
2. 提取：日期、省、市、区、地点、鸟种（中文名+拉丁名+数量）
3. 忽略：三个保护等级列
4. 鸟种匹配：拉丁名匹配 → 失败则中文名匹配
5. 导入前显示预览确认

## 五、鸟种数据文件说明

### china_birds.json

1140 条，每条格式：
```json
{ "id": 71, "s": "Dendrocygna javanica", "e": "Lesser Whistling Duck", "z": "栗树鸭", "p": "lishuya", "i": "lsy" }
```

字段含义：
- `id`: IOC 14.1 序号
- `s`: 学名
- `e`: 英文名
- `z`: 中文名
- `p`: 中文名全拼（无声调、无空格，如 `zongbeibolao`）
- `i`: 中文名拼音首字母（如 `zbbl`）

### bird_list_ioc14_1.json

11194 条全球鸟种，格式同上但多一个 `cn` 字段（1=中国鸟种，0=非中国）。目前 app 只使用中国鸟种数据，全球数据备用。

## 七、注意事项

1. 这是个人日记工具，不需要登录系统、不需要后端服务器、不需要多用户
2. 所有数据存 localStorage，务必尽早做导出功能
3. 鸟种数据文件是独立的 JSON，不要硬编码进代码里，import 进来即可
4. 搜索体验是核心：用户在野外单手操作，输入两三个字母就要能定位到目标鸟种
5. 数量输入用点击弹出输入框，不要用 +/- 按钮（野外容易误触）
6. 英文名必须独立一行显示，不要和中文名挤在同一行
7. 汇总页鸟单星号列固定宽度，保证有星和没星的鸟名对齐
8. 情绪落地模块只做占位入口，具体内容后续补充，不要自行编造引导文案
