# birdlog
鸟单自用款
> **文档用途：** 交给 AI 编程助手（Claude Code / Cursor / Codex 等）作为开发指令  
> **项目性质：** 个人日记本型观鸟记录应用，兼具正念/情绪落地功能  
> **目标用户：** 开发者本人，中国观鸟爱好者  
> **技术路线：** React 单页应用，持久化存储，后期可扩展后端  
> **鸟种名录：** IOC World Bird List v14.1 多语言版（含中文名、拼音索引）  
> **当前状态：** 四Tab框架+记录流程+数据持久化已完成，可运行原型见 `birdlog.jsx`

## 二、数据模型

### 2.1 事件（Event）

```
Event {
  id: string             // 唯一标识（生成方式：Date.now().toString(36) + random）
  date: string           // 观测日期 YYYY-MM-DD
  startTime?: string     // 开始时间（可选）
  endTime?: string       // 结束时间（可选）
  isAbroad: boolean      // 是否国外
  country?: string       // 国家（仅国外时填写）
  province?: string      // 省（仅国内，下拉选择34个省级区划）
  city?: string          // 市/区（文本输入）
  location: string       // 具体观测地点名称
  companions: string[]   // 同行人名字列表
  records: BirdRecord[]  // 鸟单
  journal: string        // 观鸟流水账（整体叙事文字）
  journalMedia: MediaRef[] // 流水账中引用的照片/音频（待实现）
  createdAt: string      // 创建时间 ISO string
}
```

### 2.2 鸟种记录（BirdRecord）

```
BirdRecord {
  birdId: number         // 对应鸟种名录 ID
  birdZh: string         // 冗余存储中文名（便于显示和搜索，不必每次查表）
  qty: number            // 数量，默认 1，点击数字直接弹出输入框编辑
  star: boolean          // 是否标星（含义："这次观察值得一记"）
  gender: "未知"|"雄"|"雌"
  behaviors: string[]    // 选中的行为标签
  customBehaviors: string[] // 用户在该记录上新建的自定义行为标签
  note: string           // 鸟种笔记
  photos: MediaRef[]     // 该鸟种的照片（待实现）
  audio: MediaRef[]      // 该鸟种的音频（待实现）
}
```

### 2.3 媒体引用（MediaRef）— 待实现

```
MediaRef {
  id: string
  type: "photo"|"audio"
  uri: string            // base64 data URL 或 blob URL
  timestamp: string
  caption?: string
}
```

### 2.4 鸟种静态数据（Bird）

内嵌在 `birdlog.jsx` 中，格式为管道符分隔文本，每行一条：

```
id|中文名|拼音首字母|全拼|英文名|学名
71|栗树鸭|lsy|lishuya|Lesser Whistling Duck|Dendrocygna javanica
```

解析后结构：
```
Bird { id: number, z: string, i: string, p: string, e: string, s: string }
```

当前收录：1140 种中国鸟种（IOC 14.1 中国区，实际约 1399 种，后续补充）。

### 2.5 朋友（Companion）

```
Companion {
  name: string           // 姓名（唯一标识）
  note?: string          // 备注（用户手动填写，如"大学同学"、"本地鸟会"等）
  photo?: string         // 头像照片（base64 data URL，用户上传）
  // 以下为自动统计，不需要手动维护：
  eventCount: number     // 一起观鸟的次数（从事件数据中计算）
  firstEventDate: string // 第一次同行日期（从事件数据中计算）
}
```

存储 key：`birdlog-companions`，格式为数组。当前原型中 companions 存储为 `string[]`（仅名字），**需要升级为对象数组**以支持备注和照片。

## 四、数据导入（待实现）

### 4.1 中国观鸟记录中心

导出格式为 TSV，字段：
```
报告编号 | 观测开始时间 | 观测结束时间 | 中文名 | 拉丁名 |
省 | 州/市 | 区/县 | 观测点名称 | IUCN受胁级别 |
国家保护等级 | CITES保护等级 | 鸟种数量
```

导入逻辑：
1. 按"报告编号"分组，每组 = 一个事件
2. 提取：日期、省、市、区、观测点名称、鸟种（中文名+拉丁名+数量）
3. 忽略：IUCN、国家保护等级、CITES
4. 鸟种匹配：先用拉丁名匹配 IOC 14.1 的 `s`（学名）字段，匹配不上则用中文名匹配 `z` 字段
5. 导入前显示预览让用户确认

## 六、技术说明

### 存储
- 当前使用 `window.storage` API（平台提供的持久化 key-value 存储）
- 存储 key：`birdlog-events`（事件数组）、`birdlog-companions`（朋友数据）
- 如迁移到独立部署，可替换为 IndexedDB 或 localStorage
- 尽早实现导出功能防止数据丢失

### 鸟种名录
- 文件独立，更换名录只需替换数据，app 代码不变
- 中国鸟种标记目前覆盖约 82%（1140/1399），后续需补充

### 拼音搜索
- 每个中文鸟名预生成：全拼（py）和首字母（ini）
- 搜索时同时匹配5个维度：中文包含 / 首字母前缀 / 全拼前缀 / 英文包含 / 学名包含

### 配色
```
背景: #faf8f5    卡片: #ffffff    主题: #4a7c59    主题浅: #e8f0e4
文字: #2c2c2c    次要: #7a7a7a    边框: #e8e4df    星标: #f59e0b
删除: #d4534b    暖底: #f5f0eb
```
